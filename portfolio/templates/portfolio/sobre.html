Arquitetura MVT do Django

O Django utiliza a arquitetura MVT (Model-View-Template).

  • Model: representa os dados e a base de dados.
  • View: contém a lógica da aplicação e processa pedidos.
  • Template: define a interface gráfica apresentada ao utilizador.

Fluxo: o utilizador faz um pedido → a view processa → o model vai à base de dados → os dados são enviados para o template → o utilizador vê a página.

Modelação

A aplicação foi modelada utilizando modelos Django definidos no ficheiro models.py. Cada modelo corresponde a uma tabela na base de dados.

Os relacionamentos entre entidades são feitos com ForeignKey e ManyToMany.
